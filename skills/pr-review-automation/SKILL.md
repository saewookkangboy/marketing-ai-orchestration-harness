# PR Review Automation

## Purpose

오픈된 PR의 리뷰 코멘트를 수집·분류·반영해, 머지 가능한 상태로 빠르게 정리한다.

## Triggers

- "PR 코멘트 반영해줘"
- "리뷰 피드백 처리해줘"
- "열린 스레드 정리해줘"
- "수정하고 답글 초안까지 만들어줘"

## Inputs

- 필수: PR 식별자(URL, 번호, 현재 브랜치)
- 필수: 반영 범위(전체/특정 리뷰어/특정 파일)
- 선택: 우선순위 기준(critical first, oldest first)
- 선택: 답글 톤(간결/상세)

## Workflow

1. PR 상태 확인
   - 리뷰 코멘트, 열린 스레드, 변경 파일을 수집한다.
2. 피드백 분류
   - 버그/안전/성능/스타일/질문으로 태깅하고 우선순위를 매긴다.
3. 반영 실행
   - 명확한 수정사항은 즉시 코드 반영 후 로컬 검증을 수행한다.
4. 답글 초안 생성
   - 각 스레드별로 "무엇을 반영했는지/왜 보류했는지"를 짧게 작성한다.
5. 최종 점검
   - 남은 열린 스레드와 테스트 상태를 확인하고 후속 작업을 제안한다.

## Safety Rules

- 사용자 승인 없이 `push --force` 금지
- 보안·데이터 무결성 관련 코멘트는 무조건 우선 처리
- 요구사항이 불명확한 코멘트는 추정 구현하지 않고 질문으로 승격

## Output Contract

- 리뷰 처리 요약(카테고리별 반영/보류 건수)
- 수정 파일 목록
- 로컬 검증 결과(실행 커맨드와 핵심 결과)
- 스레드 답글 초안(필요 시)

## Example Invocation

"현재 브랜치 PR의 열린 리뷰 코멘트를 critical 우선으로 반영하고, 각 스레드 답글 초안까지 작성해줘."
