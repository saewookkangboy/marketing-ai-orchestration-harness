# Automation Skill Generator (Claude Cowork + OpenAI Codex)

## 목적

사용자 요청을 받아 **자동화 가능한 스킬 문서(`SKILL.md`)**를 생성/개선한다.  
이 스킬은 Claude Cowork, OpenAI Codex 환경에서 공통으로 재사용 가능한 구조를 우선한다.

## 언제 사용하나

- "자동화 스킬 만들어줘"
- "기존 프롬프트를 Skill.md 형태로 바꿔줘"
- "Codex/Claude 둘 다 돌 수 있게 스킬 템플릿화해줘"
- 반복 실행되는 작업(리뷰, 배포, 리포트, QA, 데이터 정리)을 문서화하고 싶을 때

## 입력 요구사항

아래 항목을 우선 수집한다.

1. 작업 목표: 무엇을 자동화할지
2. 실행 환경: 로컬/CI, 사용 도구(예: gh, python, node, MCP)
3. 제약사항: 금지 명령, 승인 필요 단계, 보안 요구
4. 성공 기준: 완료 조건, 품질 게이트, 실패 시 처리
5. 산출물 형식: 파일/PR/리포트/코멘트/체크리스트

정보가 부족하면 추정하지 말고 최소 질문으로 보강한다.

## 생성 규칙

1. **결과 중심 서술**: "무엇을 만들었는가"보다 "왜/언제 쓰는가"를 먼저 쓴다.
2. **실행 가능한 단계화**: 각 단계를 입력/행동/검증으로 분리한다.
3. **안전 우선**: 파괴적 명령 금지, 비밀정보 처리, 승인 포인트를 명시한다.
4. **플랫폼 중립**: 특정 모델 전용 표현 대신 공통 지시문을 우선 사용한다.
5. **관측 가능성**: 로그/리포트/체크 포인트를 정의해 재현 가능하게 만든다.

## 출력 형식

최종 출력은 아래 구조를 갖는 `SKILL.md` 초안을 생성한다.

```md
# <Skill Name>

## Purpose
<한 줄 목적>

## Triggers
- <이 스킬이 필요한 사용자 표현 3~8개>

## Inputs
- <필수 입력>
- <선택 입력>

## Workflow
1. <준비 단계>
2. <핵심 실행 단계>
3. <검증 단계>
4. <마무리/보고 단계>

## Safety Rules
- <금지/주의/승인 포인트>

## Output Contract
- <결과물 형식, 파일명, 보고 규칙>

## Example Invocation
<짧은 실행 예시>
```

## 품질 체크리스트

- 목적/트리거/입력/출력 계약이 서로 모순 없는가
- 단계마다 검증 포인트가 있는가
- 실패 경로(재시도, 롤백, 에스컬레이션)가 정의됐는가
- 민감정보/권한 관련 주의사항이 포함됐는가
- Claude/Codex 어느 쪽에서도 읽어도 실행 의도가 동일한가

## 빠른 파생 템플릿

아래 4가지 카테고리로 빠르게 파생 생성할 수 있다.

1. Review Automation: PR 코멘트 수집/분류/반영 자동화
2. CI Fix Automation: 실패 체크 탐지→로그 요약→수정 루프
3. Release Automation: 버전 태깅/체인지로그/배포 확인
4. Report Automation: 실행 로그 기반 주간/월간 보고서 생성

요청이 오면 위 카테고리를 우선 매핑한 뒤, 도메인 특화 단계만 추가한다.
