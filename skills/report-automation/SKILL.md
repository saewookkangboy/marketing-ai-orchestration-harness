# Report Automation

## Purpose

실행 로그와 결과 데이터를 기반으로 주간/월간 운영 보고서를 자동 생성한다.

## Triggers

- "이번 주 작업 보고서 만들어줘"
- "실행 결과 요약 리포트 만들어줘"
- "운영 지표 정리해줘"
- "의사결정용 한 페이지 보고서로 정리해줘"

## Inputs

- 필수: 기간(주간/월간/사용자 지정)
- 필수: 데이터 소스(로그, 산출 JSON, 커밋, 이슈)
- 필수: 대상 독자(실무/리드/임원)
- 선택: 강조 지표(KPI, 품질, 속도, 리스크)

## Workflow

1. 데이터 수집
   - 기간 내 결과물과 근거 데이터를 모은다.
2. 정규화/클린업
   - 중복·노이즈를 제거하고 단위/기준을 맞춘다.
3. 핵심 인사이트 추출
   - 성과, 병목, 리스크, 다음 액션을 우선순위로 정리한다.
4. 보고서 렌더링
   - 대상 독자에 맞춰 요약 길이와 용어 수준을 조정한다.
5. 검증
   - 수치 합산, 링크 무결성, 근거 추적 가능성을 점검한다.

## Safety Rules

- 수치/지표는 추정치와 확정치를 구분한다.
- 근거 없는 단정 문장을 쓰지 않는다.
- 민감정보(개인/비공개 KPI)는 마스킹 또는 제외한다.

## Output Contract

- 보고서 본문(요약, 상세, 리스크, 다음 액션)
- 핵심 KPI 스냅샷(기간 비교 포함)
- 근거 링크/파일 목록
- 실행 가능한 후속 TODO

## Example Invocation

"최근 2주 기준으로 마케팅 오케스트레이션 실행 결과를 리드용 보고서 형태로 작성하고, 다음 스프린트 액션 아이템까지 정리해줘."
