# Release Automation

## Purpose

릴리즈 준비부터 태그/노트/배포 확인까지 일관된 절차로 자동화한다.

## Triggers

- "릴리즈 준비해줘"
- "체인지로그 만들고 배포해줘"
- "버전 태그 찍고 릴리즈 노트 작성해줘"
- "프리뷰/프로덕션 배포 상태 확인해줘"

## Inputs

- 필수: 릴리즈 대상 브랜치/커밋
- 필수: 버전 정책(semver, 날짜 버전 등)
- 필수: 배포 환경(프리뷰/스테이징/프로덕션)
- 선택: 릴리즈 노트 포맷, 포함/제외 범위

## Workflow

1. 릴리즈 전 게이트 확인
   - 테스트/린트/빌드/필수 체크 통과 여부를 점검한다.
2. 변경점 집계
   - 커밋/PR/이슈를 기준으로 사용자 영향 중심으로 요약한다.
3. 버전 및 태그 처리
   - 버전 번호를 확정하고 태그 또는 릴리즈 엔트리를 생성한다.
4. 배포 실행
   - 대상 환경으로 배포하고 배포 식별자(URL/릴리즈 ID)를 수집한다.
5. 사후 검증
   - 핵심 헬스체크와 회귀 포인트를 확인한다.
6. 공지/기록
   - 릴리즈 노트와 롤백 지점을 명확히 남긴다.

## Safety Rules

- 프로덕션 배포는 승인 없이 실행하지 않는다.
- 롤백 경로 없는 릴리즈는 진행하지 않는다.
- 마이그레이션 포함 시 데이터 무결성 확인 항목을 필수로 둔다.

## Output Contract

- 릴리즈 버전/태그/배포 ID
- 릴리즈 노트(요약 + 영향 범위 + 주의사항)
- 검증 체크 결과
- 롤백 기준과 절차

## Example Invocation

"main 기준으로 이번 변경을 semver patch 릴리즈로 준비하고, 배포 검증까지 포함한 릴리즈 노트를 만들어줘."
